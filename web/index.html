🔑 Connecting Your Private Infrastructure

Since this hosted dashboard is running on a public cloud server, it operates in Simulated Mode (it has no access to your private local networks). To deploy actual containers on your own local hosts or remote clusters, use one of the following methods:

Option A: Local Binary Daemon (Recommended) Run DoktriAI on your private host. The orchestrator will automatically bind to your local socket: # Run locally in your terminal
go run ./cmd/doktriai-api -addr :18080 Once running, the local dashboard at localhost:18080 will read and deploy container processes directly to your local Docker daemon socket.
Option B: Remote SSH / TCP Tunneling Expose your private Docker daemon securely via SSH or TLS TCP sockets, and configure your target environment settings: # Override target Docker Host URL
export DOCKER_HOST="tcp://your-private-ip:2376" Attach client TLS certificates in ~/.docker/ or specify SSH keys so the cloud api can invoke container workloads remotely.

workspace://documentation

Complete developer reference: server flags, REST API endpoints, MCP agent tools, WorkloadSpec schema, OPA security policy, and runnable cURL/CLI examples — all sourced from the live control-plane codebase.

1. Health diagnostics probe

Check liveness status. Returns status string indicating internal database/driver connectivity.

# Send a health check request to check server status
curl http://localhost:18080/api/health

2. List Active Workload Specifications

Fetch all declared applications and active running containers. Requires authentication.

# Fetch all active specs and runtime workloads
curl -H "X-Doktri-Role: admin"      http://localhost:18080/api/workloads

3. Deploy Container App (Upsert)

Deploy a new container or update an existing one. Pass key properties like image, replicas, and ports.

# Deploy / update a container using standard REST API
curl -X POST http://localhost:18080/api/workloads   -H "Content-Type: application/json"   -H "X-Doktri-Role: admin"   -H "X-Doktri-Actor: alice"   -d '{
    "name":           "hello-web",
    "image":          "nginx:alpine",
    "replicas":       2,
    "port":           8080,
    "containerPort":  80,
    "deployStrategy": "rolling",
    "env":            {"APP_ENV": "production"},
    "resources":      {"memoryMb": 128, "cpuShares": 512}
  }'

4. Spec Dry-run Validation

Validate format and verify permissions against policy without deploying to runtime.

# Dry-run validation check
curl -X POST http://localhost:18080/api/validate   -H "Content-Type: application/json"   -H "X-Doktri-Role: operator"   -d '{"name":"test-app","image":"ubuntu:22.04","replicas":1,"port":9000,"containerPort":9000}'

5. Tail Container Logs

Get raw output streams from the running containers of a specified workload.

# Tail stdout/stderr stream from container
curl -H "X-Doktri-Role: reader"      "http://localhost:18080/api/logs/hello-web?tail=30"

6. Acquire Reconciler Lock

Lock control plane to stop background reconciliation loop operations for manual repairs.

# Acquire global control plane lock
curl -X POST http://localhost:18080/api/lock   -H "Content-Type: application/json"   -H "X-Doktri-Role: admin"   -H "X-Doktri-Actor: alice"   -d '{"reason":"planned maintenance window"}'

7. Release Reconciler Lock

Unlock control plane and resume automatic reconciliation loop drift corrections.

# Release the control plane lock
curl -X DELETE http://localhost:18080/api/lock   -H "X-Doktri-Role: admin"

8. Force Reconcile Trigger

Execute a reconciliation run instantly instead of waiting for the background tick.

# Trigger immediate loop run
curl -X POST http://localhost:18080/api/reconcile   -H "X-Doktri-Role: admin"

9. Query Pending PTE plans

Retrieve a list of high-risk plans currently stuck in the Plan-To-Execute queue.

# Fetch all plans waiting for approval
curl -H "X-Doktri-Role: admin"      http://localhost:18080/api/plan

10. Approve Pending PTE Plan

Authorize a paused operation (e.g. scale > 5) and execute the configuration changes.

# Approve a plan by plan ID
curl -X POST http://localhost:18080/api/plan/PLAN-ID-HERE/approve   -H "X-Doktri-Role: admin"

11. Rollback Version

Restore configuration metadata parameters back to a specific version number.

# Rollback workload to version 2
curl -X POST http://localhost:18080/api/workloads/hello-web/rollback   -H "Content-Type: application/json"   -H "X-Doktri-Role: admin"   -d '{"version":2}'

12. MCP tools list discovery

Discover available tools for AI agents (Claude, GPT, Gemini) using JSON-RPC standard.

# Get MCP server tools spec
curl -X POST http://localhost:18080/api/mcp   -H "Content-Type: application/json"   -H "X-Doktri-Role: admin"   -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'

# DOKTRIAI Python client examples
import requests, json

BASE  = "http://localhost:18080"
HDRS  = {"X-Doktri-Role":"admin", "X-Doktri-Actor":"alice"}

# 1 - Deploy workload
spec = {
  "name":          "api-service",
  "image":         "myapp:v2.1.0",
  "replicas":      2,
  "port":          3000,
  "containerPort": 3000,
  "deployStrategy":"rolling",
  "maxSurge":      1,
  "maxUnavailable":0,
  "env":           {"APP_ENV":"production"},
  "resources":     {"memoryMb":256, "cpuShares":1024},
  "healthCheck":   {"httpPath":"/health", "httpPort":3000, "intervalSec":15}
}
r = requests.post(f"{BASE}/api/workloads", json=spec, headers=HDRS)
print(r.json())

# 2 - List all workloads
wls = requests.get(f"{BASE}/api/workloads", headers=HDRS).json()
for w in wls:
    print(w["spec"]["name"], w["healthy"], len(w["actual"]),"containers")

# 3 - Validate spec without deploying
res = requests.post(f"{BASE}/api/validate", json=spec, headers=HDRS)
print("valid:", res.status_code == 200)

# 4 - Get logs
logs = requests.get(f"{BASE}/api/logs/api-service",
                    params={"tail":100}, headers=HDRS)
print(logs.text)

# 5 - Acquire reconciler lock
requests.post(f"{BASE}/api/lock",
              json={"reason":"db migration"}, headers=HDRS)

# 6 - MCP: agent calls deploy_workload
mcp_body = {
  "jsonrpc":"2.0", "id":1,
  "method":"tools/call",
  "params":{
    "name":"deploy_workload",
    "scope":"list_workloads,deploy_workload",
    "goal":"Scale api-service to production",
    "arguments": spec
  }
}
mcp_r = requests.post(f"{BASE}/api/mcp", json=mcp_body, headers=HDRS)
result = mcp_r.json()["result"]
if result["status"] == "pending_approval":
    print("PTE plan created:", result["planId"])
else:
    print("Deployed:", result["status"])

Go net/http client implementation

Use standard library net/http structs to execute declarative JSON requests, fetch workload parameters, and manage lock mutexes natively.

// DOKTRIAI Go client examples
package main

import (
  "bytes"
  "encoding/json"
  "fmt"
  "io"
  "net/http"
)

const base = "http://localhost:18080"

func doktriReq(method, path string, body any) []byte {
  var buf *bytes.Buffer
  if body != nil {
    b, _ := json.Marshal(body)
    buf = bytes.NewBuffer(b)
  } else {
    buf = &bytes.Buffer{}
  }
  req, _ := http.NewRequest(method, base+path, buf)
  req.Header.Set("Content-Type", "application/json")
  req.Header.Set("X-Doktri-Role", "admin")
  req.Header.Set("X-Doktri-Actor", "alice")
  resp, _ := http.DefaultClient.Do(req)
  defer resp.Body.Close()
  data, _ := io.ReadAll(resp.Body)
  return data
}

func main() {
  // 1 - Deploy workload with rolling strategy + health probe
  spec := map[string]any{
    "name":           "api-service",
    "image":          "myapp:v2.1.0",
    "replicas":       2,
    "port":           3000,
    "containerPort":  3000,
    "deployStrategy": "rolling",
    "maxSurge":       1,
    "maxUnavailable": 0,
    "env":            map[string]string{"APP_ENV": "production"},
    "resources":      map[string]any{"memoryMb": 256, "cpuShares": 1024},
    "healthCheck":    map[string]any{
      "httpPath":    "/health",
      "httpPort":    3000,
      "intervalSec": 15,
    },
  }
  fmt.Println(string(doktriReq("POST", "/api/workloads", spec)))

  // 2 - Validate spec (dry-run)
  fmt.Println(string(doktriReq("POST", "/api/validate", spec)))

  // 3 - Get workload status
  fmt.Println(string(doktriReq("GET", "/api/workloads/api-service", nil)))

  // 4 - Rollback to version 1
  fmt.Println(string(doktriReq("POST", "/api/workloads/api-service/rollback",
    map[string]any{"version": 1})))

  // 5 - MCP deploy via agent
  mcp := map[string]any{
    "jsonrpc": "2.0", "id": 1,
    "method":  "tools/call",
    "params": map[string]any{
      "name":      "deploy_workload",
      "scope":     "list_workloads,deploy_workload",
      "goal":      "Scale api-service",
      "arguments": spec,
    },
  }
  fmt.Println(string(doktriReq("POST", "/api/mcp", mcp)))
}

3. MCP Agent Tool Reference

AI agents (Claude, GPT, Gemini) interact over the Model Context Protocol (MCP) endpoint at POST /api/mcp. All calls use a JSON-RPC 2.0 envelope: {"jsonrpc":"2.0","method":"tools/call","params":{...}}

Tool Name PTE Gated? Description & Arguments
tools/list No Returns the schema of all callable tools with descriptions and approval flags.
deploy_workload Yes* Creates or updates a workload. Args = full WorkloadSpec JSON. If replicas > 5 or high-risk image, returns pending_approval status with a planId.
list_workloads No Returns array of WorkloadStatus objects with desired spec and actual containers.
get_workload No Args: {"name":"hello-web"}. Returns single workload status or 404 error.
delete_workload Always Args: {"name":"hello-web"}. All autonomous deletions route through PTE Gate. Returns pending_approval.
get_logs No Args: {"name":"hello-web","tail":50}. Returns last N lines from container stdout.
list_pending_plans No Returns all PTE plans awaiting human approval. Use before calling approve_plan.
approve_plan No Args: {"id":"plan-uuid"}. Marks plan approved and immediately applies the stored spec.
reject_plan No Args: {"id":"plan-uuid","comment":"reason"}. Permanently rejects the plan.
get_behavior_metrics No Returns per-actor anomaly counters for security dashboards.
🔐 MCP Scope Claim Agents must declare their scope in the params.scope field, e.g. "scope": "list_workloads,deploy_workload". Any tool call outside the declared scope is rejected by ValidateAgentScope() before execution.

4. WorkloadSpec Field Glossary

Every deploy call (REST or MCP) requires a WorkloadSpec JSON body. These are the exact struct fields from doktriai-packages/types.go:

Field (JSON key) Type Meaning & Valid Values
name string Unique workload identifier. Lowercase alphanumeric + hyphens. e.g. nginx-web.
image string Docker image reference: nginx:alpine, redis:7.2, or fully-qualified digest.
replicas integer Desired container count. Range 1-50. Values > 5 trigger PTE Gate automatically.
port integer Host port exposed on the Docker network (0 = no port binding).
containerPort integer Port the process listens on inside the container. Typically matches port.
runtime string "docker" (default). Future: "kubernetes".
deployStrategy string "recreate" (default) - stop all, start new. "rolling" - replace one-at-a-time with zero downtime.
maxSurge integer Max extra containers allowed above replicas during rolling update.
maxUnavailable integer Max containers that may be stopped simultaneously during rolling update.
securityMode string Validation tier: "dev" | "staging" | "production". Production enforces registry allowlist + digest pinning.
env map[str]str Environment variables injected into containers. Values max 4096 chars. No SECRET_ prefix in dev.
resources.cpuShares int64 Relative CPU weight. 512 = half a core, 1024 = 1 core. Docker default: 1024.
resources.memoryMb int64 Memory limit in megabytes. 0 = unlimited. e.g. 256 = 256 MB cap.
healthCheck.httpPath string HTTP GET path for health probe, e.g. /healthz. Reconciler marks container unhealthy if probe fails.
healthCheck.intervalSec int Seconds between liveness checks. Default 30.
volumes[].hostPath string Absolute path on the host to bind-mount into the container.
volumes[].containerPath string Mount target path inside the container.
labels map[str]str Key-value labels attached to Docker containers (for filtering/grouping). e.g. {"team":"platform"}.

6. Key Terms & Acronyms

Control Plane The central system (this server) that maintains desired state and continuously drives actual runtime state to match it.
Reconciler Loop Background goroutine that ticks every -reconcile-every interval, compares spec vs actual, and applies corrections (start/stop containers).
When actual container count differs from replicas in the spec. The reconciler auto-heals drift within one loop tick.
OPA (Open Policy Agent) Policy-as-code engine. DOKTRIAI uses an OPA YAML policy file to define role permissions and declare which images are allow-listed.
PTE Gate Plan-To-Execute Gate. High-risk operations are paused here; an Admin must approve the plan before execution proceeds.
MCP Model Context Protocol. JSON-RPC 2.0 bridge that lets AI agents call control-plane operations as typed tool calls.
SSE Server-Sent Events. Long-lived HTTP stream (GET /api/events) that pushes real-time control-plane events to the browser.
Behavioral Tracker Sliding-window anomaly engine. Tracks deploy/delete/reject rates per actor in a 5-minute window and flags suspicious patterns.
JWT JSON Web Token. Signed token used for agent authentication. Verified by core.VerifyAgentJWT().
HMAC Token Hash-based request token for short-lived CLI authentication. Verified by core.VerifyRequestToken().
Rolling Strategy Replaces containers one-at-a-time during updates. Controlled by maxSurge (extra containers) and maxUnavailable (offline limit).
OTEL / OpenTelemetry Distributed tracing framework. Each MCP tool call and reconcile tick emits a span to Jaeger/OTLP. Disable with DOKTRIAI_OTEL_DISABLED=true.
