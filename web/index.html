workspace://security

DOKTRIAI 5-Layer Agentic Security Control Plane — monitoring PTE approvals, behavioral anomalies, audit trail integrity, and supply chain policies in real time.

Layer 2

Pending Approvals (Plan-Then-Execute Gate)

High-risk agent actions (replicas > 5, sensitive env keys, all deletes) are held here until a human operator approves or rejects them. Unapproved plans expire after 15 minutes. Prevents ASI09 — autonomous AI damage.

Loading pending approvals...
Layer 3

Behavioral Anomaly Detection

Rolling 5-minute window tracks deploy/delete/reject rates per actor. Actors exceeding thresholds are flagged and emit anomaly events. Detects ASI10 — rogue agents.

Loading behavioral metrics...
Layer 3

Security Audit Trail (SHA256 State Diff Hashing)

Every state mutation records a SHA256 hash of desired state before and after. Signed tokens show ✓ signed badge. Approved plans show ✓ approved badge. Prevents ASI06/ASI10 — audit log poisoning.

Security & PTE Approval Gate

Layer 2 — PTE Approval Gate Security · Required High-risk deploys (replicas > 5, sensitive keys, all deletes) require human approval. Navigate to Security → Pending Approvals to approve or reject them before they execute.
Layer 3 — Behavioral Anomaly Security · Detection Rolling 5-minute windows track deploy/delete/reject rates per actor. Actors exceeding thresholds are flagged and emit anomaly events visible in the Security panel.
Control Plane Lock Security · Lock Use Reconciler → Control Plane Lock to freeze all automated deployment changes. Enter a reason and click Acquire Lock. Release when safe to resume operations.

MCP Agent Integration

Connecting an AI Agent via MCP Integration · AI Agents The MCP server exposes tools over JSON-RPC 2.0 at POST /api/mcp. Available tools:
tools/list — list all agent-callable tools
deploy_workload — deploy a container via agent
list_workloads — fetch current workload state
delete_workload — decommission a workload
get_logs — stream container logs
Generate a signed HMAC token in CLI Reference → Secure Token Generator and attach it as X-Doktri-Token header in agent calls.

Phase 2 Upgrades & Advanced Operations

① YAML Manifest Deployments Operation · YAML Editor Deploy declaratively using the YAML Editor on the Workloads page:
• Paste your YAML manifest block.
• Click Deploy Manifest.
• Supports complete attributes like deployStrategy: rolling, maxSurge, etc.
② Rolling Upgrade Strategy Operation · Upgrades Upgrade container workloads sequentially using the rolling strategy. Reconciler replaces replicas one-by-one with zero-downtime, keeping active traffic routed. Set deployStrategy: rolling.
③ Visual Database Rollbacks Operation · DB History Look up full workload database version snapshots:
• Find your workload in the status table.
• Click the new history button.
• Select a previous version and click Rollback.
④ Cryptographic JWT Keys Operation · Security Generate stateless JWT authentication keys:
• Go to CLI Reference tab.
• Select Token Type: SaaS Agent JWT.
• Specify Actor Name & Agent Scope and click Generate Token.

Control Plane Execution Flow Guide

⓵ Input Trigger

Developer or AI Agent pushes deployment intent via Web UI (YAML Editor), CLI command, or MCP JSON-RPC protocol.

⓶ Gateway & Auth

doktriai-api interceptor validates signatures, verifies stateless JWT claims, and logs actions to the secure audit trail.

⓷ State Reconciler

Core engine validates schema and screens spec against doktri-policy.yaml rules (registry allowlist, replica thresholds).

⓸ Database Store

Core runs reconciliation loop, persisting version snaps in DB history and detecting drift from desired configuration.

⓹ Runtime Drivers

Virtualization drivers (Docker host socket/Kubernetes operator) execute upgrades (rolling vs recreate strategies) to achieve target state.
